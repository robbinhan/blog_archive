ssh连接

發表於 2013-08-11

在我们最初装好ssh后，连接某台服务器我们需要输入一长串的命令（ssh username@host等 ），然后连上了还要输入password。所以我希望是做到 ssh servername就能自动连上，并且不需要输入password。

首先我们在自己机器上生成一个私钥和公钥(私钥必须chmod 600)，将带.pub后缀的公钥传到远程主机的~/.ssh/目录下，然后将其内容cat进~/.ssh/authorized_keys文件；然后在我们自己机器上编辑~/.ssh/config文件；


例：

Host servername

   port 22

   user username

   hostip ip

   IdentityFile 私钥的路径（名字可以和公钥不同）


配置完成后应该就可以直接ssh servername连接了，如果你发现远程主机已经有一个pub文件和私钥了，则可以将其私钥的内容拷贝过来保存，然后添加配置一样可以连接。

PHP中到底用数组还是用对象

發表於 2013-08-11

   在大部分语言中，我们写代码基本都是用对象的形式保存数据，或者相互传递；但是PHP这个语言的数组太强大了，所以大多数时候我们会选择数组来进行code，感觉一般用途数组也很好，没必要再用对象。

   没错，我认为写一个单独的或几个文件的脚本直接用数组确实很方便，没什么问题，但是到了项目上是否这样也会好？可能初期的时候或者这个项目只有你一个人写的时候没有问题，但是越到后期，或者中途有人参与进来（虽然这是不推荐的，但现实工作中确实存在这样的情况），你会发现用数组来传递数据会有这样那样的问题，比方可能会遇到命名的问题等，可能都是小问题只要稍微改改就能避免，但是时间越长，小问题会变的越来越大，可能就会导致重构提前了。

   所以，我个人认为，尽量还是使用对象，这样代码看上去也容易理解点，而在一些独立脚本中可以适当的书用数组。

mysql tinyint(3)

發表於 2013-08-11

tinyint(3)不是代表长度是3个字符的整数，他的范围是无符号是0-127，有符号是-255 – 255

mysql中所有整形字段后面的数字都只是表示最终数据显示的长度，而与字段类型本身长度无关。
