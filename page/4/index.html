Ubuntu 下编译PHP源码

發表於 2013-08-11

PHP版本：5.3.3

1.确保以安装build-essential,如果没有则执行 sudo apt-get install build-essential

2.执行 tar -zxvf php-5.3.3.tar.bz2

3.cd php-5.3.3

4. ./buidconf

   如果出现一下错误： You should not run buildconf in a release package. use buildconf –force to override this check.

   执行 ./buildconf –force

   可能会提示： Forcing buildconf buildconf: checking installation… buildconf: autoconf not found. You need autoconf version 2.13 or newer installed to build PHP from SVN. make: * [buildmk.stamp] 错误 1

   则说明没有安装autoconf，需要安装再执行buildconf，但是必须安装2.13版，如果从源上安装高版本肯能会报： Forcing buildconf buildconf: checking installation… buildconf: autoconf version 2.65 (ok) buildconf: Your version of autoconf likely contains buggy cache code. Running vcsclean for you. To avoid this, install autoconf-2.13. Can’t figure out your VCS, not cleaning.

   从autoconf－2.13下载，然后执行

   tar -xzf autoconf-2.13.tar.gz

   cd autoconf-2.13

  ./configure

   make

   make install

   安装完成后执行： ./buildconf –force 成功！

   然后执行： ./configure 如果没有安装libxml2，可能会提示需要安装，

   执行： sudo apt-get install libxml2-dev

   然后再执行./configure 成功。 执行make

  ok!结束!! make test测试下吧

我对网瘾的思考

發表於 2013-08-11

    今天在看《拆掉思维里的墙》一书，看到第二章（有趣的生命扑面而来）时，突然想到为什么很多人会有“网瘾”。个人认为“网瘾”这个概念不单是指喜欢网络游戏上瘾，也包括泡BBS，SNS平台，包括现在的微博。大部分“网瘾”的人都是年轻人，我觉得年轻人有个特点，这是从青春期或更小的时候就有的特点，就是好奇心。我们喜欢新鲜的东西，而互联网就是能满足我们好奇心的地方，每分钟或者每秒钟都会有新的东西产生来满足我们的好奇心，所以可能就在我们对其失去兴趣的那一瞬间，新的事物又让我们再次产生了兴趣。循环往复我们就上“瘾”了。    而为什么会喜欢上网，而不喜欢天天对着教科书呢。其实仔细想想，当我们每学期开始看到新的教科书，看到那些新开的课程，我们的心理也许当时确实时有那么一种冲动，希望早点能上这门课，早点听到关于这门课程的讲述。但是当我们上了几节课后，我们渐渐失去了兴趣，我们觉得无聊，觉得其实不是我们想象中的那么有趣，因为可能我们对于一篇课文需要读好几遍，背好几遍。对于同一类型的习题我们需要做好几遍，早就失去了最初的兴趣。     所以我认为，“网瘾”的戒除不是靠强制性的措施，而是需要引导方式，让其接触更多的东西，让其发现真正感到有趣的东西。
