ssh连接

Veröffentlicht am 2013-08-11

在我们最初装好ssh后，连接某台服务器我们需要输入一长串的命令（ssh username@host等 ），然后连上了还要输入password。所以我希望是做到 ssh servername就能自动连上，并且不需要输入password。

首先我们在自己机器上生成一个私钥和公钥(私钥必须chmod 600)，将带.pub后缀的公钥传到远程主机的~/.ssh/目录下，然后将其内容cat进~/.ssh/authorized_keys文件；然后在我们自己机器上编辑~/.ssh/config文件；


例：

Host servername

   port 22

   user username

   hostip ip

   IdentityFile 私钥的路径（名字可以和公钥不同）


配置完成后应该就可以直接ssh servername连接了，如果你发现远程主机已经有一个pub文件和私钥了，则可以将其私钥的内容拷贝过来保存，然后添加配置一样可以连接。

          Veröffentlicht am
          
            2013-08-11
          
        

        

        
          
        
      

    


    


      
      

      
        
          
这两天研究了下rails的migration，他可以通过命令行几个简单的参数创建一个数据库表，并且整合上ActiveRecord可以灰常简单快速的完成开发。
当需要对已经创建的表做修改时需要新建一个migration，旧的类会因为版本太低不会被执行，每一次的rake db:migrate只会执行最新版的类，但是在类中可以对类本身的数据表，或其他已存在的表做任何表结构的操作。
具体看guide：http://guides.rubyonrails.org/migrations.html
还有migration的apihttp://api.rubyonrails.org/classes/ActiveRecord/Migration.html
